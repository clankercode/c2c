# Ideas (not yet ingested)

This file is for ideas we haven't processed yet. Mostly it will be Max posting stuff here for processing later. Ideas should be handled intelligently, and ideas/input sought from agents. Make it a communal process.

It should be committed to git regularly so it's obvious when there's new stuff to ingest.

## Statuses

- `new` — just arrived, not yet discussed
- `brainstorming and planning` — swarm is exploring shape; comments being gathered
- `ingested` — moved into active scope (added to `todo.txt` / `todo-ongoing.txt` / a design doc); can be deleted from here

## Handling process

1. **coordinator1 sweeps this file** at each sitrep — promotes `new` items to `brainstorming and planning` by starting a short discussion (DM/room) with relevant peers.
2. **Authors add context**: each idea should gain a few bullets (use-case, rough scope, blockers) as agents discuss.
3. **Promote to ingested**: when the idea has a concrete home (a todo entry, a design doc, a project in `todo-ongoing.txt`), mark `ingested` and remove from this file (preserved in git history).
4. **Kill ideas that don't fit**: if the swarm decides an idea is out of scope, mark `ingested` with a note explaining why (so it doesn't re-surface).

```

Submitted by: <alias> · <UTC timestamp>
Status: new | brainstorming and planning | ingested
Related: <links to todos, findings, design docs as discussion accretes>

<context, details, open questions>
```
