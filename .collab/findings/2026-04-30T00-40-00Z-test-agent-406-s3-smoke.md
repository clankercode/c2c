**`test_deliver_script_present`** — FAILED ✓ (caught by test)

```
c2c_deliver_inbox.py --help failed:
  ModuleNotFoundError: No module named 'c2c_registry'
```

**Root cause**: The import chain is:
```
c2c_deliver_inbox.py → c2c_poll_inbox.py → c2c_mcp.py → c2c_registry.py
```
`c2c_mcp.py` was added to the `.dockerignore` exceptions in S3, but `c2c_registry.py` was NOT. So `c2c_registry.py` is excluded from the Docker build context and doesn't make it into the image.

**Fix**: Add `!c2c_registry.py` to the `.dockerignore` exceptions block:
```python
# Exception: Python files needed by Dockerfile.agent runtime stage
!c2c_deliver_inbox.py
!c2c_poll_inbox.py
!c2c_poker.py
!c2c_mcp.py
!c2c_registry.py    # ← MISSING
```

## Note on methodology

The `--force-test-env` flag was needed because the test host has 2 MCP servers running (the test harness limit is 1). The compose `--build` step times out at 300s inside pytest because the OCaml build takes too long in the pytest timeout window. This is the same systemic issue as #406 S1 and S2 — the `setUpClass --build` approach is incompatible with the pytest 300s timeout for an OCaml build. The S3 smoke was run with `compose up -d` (without `--build`) using pre-built images.
