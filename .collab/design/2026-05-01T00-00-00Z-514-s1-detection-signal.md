# #514: MCP Self-Restart on Stale Broker Root

## Problem Statement

The c2c MCP server resolves its broker root via `C2c_repo_fp.resolve_broker_root()`
at startup. The parent `c2c start` process also resolves it before spawning the MCP
server child, and passes the result via the `C2C_MCP_BROKER_ROOT` environment variable.

When `C2C_MCP_BROKER_ROOT` is **not set** (i.e. the parent resolved the default path
based on the git fingerprint), the MCP server uses that env value directly. If the
git remote URL changes between when the parent started and when the MCP server was
spawned, or if the MCP server process inherits a stale `C2C_MCP_BROKER_ROOT` from
a parent whose git context differs from its own, the MCP server writes to a
different broker directory than what it would compute fresh — splitting registration
and inbox from peers.

Agents can also be started with `c2c start` from a git clone whose `remote.origin.url`
has changed since the last run, causing the same split-brain.

## Slice 1 (S1) — Detection Signal

**Committed:** `87ad5415` — peer-PASS: cedar-coder ✅

### Algorithm

In `run_inner_server ~broker_root` in `c2c_mcp_server_inner.ml`:

1. Compute `canonical = C2c_repo_fp.resolve_broker_root ()`.
2. Read `from_env = Sys.getenv_opt "C2C_MCP_BROKER_ROOT"`.
3. If `from_env = Some v` (explicit override set), skip — trust the override.
4. If `from_env = None` and `broker_root <> canonical`:
   - Log warning with inherited vs canonical paths.
   - Exit with code **42** (signals parent to re-launch with correct value).

### Exit Code 42

Chosen as non-conflicting. The outer `c2c start` catches 42 in `run_outer_loop`
and re-launches with a freshly-resolved `C2C_MCP_BROKER_ROOT`.

### Edge Cases

- **`C2C_MCP_BROKER_ROOT` explicitly set**: skip check — intentional override.
- **Nested `c2c start`**: inner MCP server sees `from_env = Some(...)` → skip.
- **Standalone `c2c-mcp-server` binary**: detection identical, exits 42 on mismatch.

### Files Changed (S1)

```
ocaml/server/c2c_mcp_server_inner.ml  — check_stale_broker_root + exit 42
```

## Slice 2 (S2) — Restart Path

**Committed:** `4a33ef71` — pending peer-PASS

### Algorithm

In `run_outer_loop` in `c2c_start.ml`, after `wait_for_child()` returns with `exit_code = 42`:

1. **Stop sidecars**: `stop_sidecar !wire_pid`, `!deliver_pid`, `!poker_pid`.
2. **Capture orphan inbox**: `capture_orphan_inbox_for_restart` — persists any messages
   that arrived during the restart gap to a pending-replay file.
3. **Compute fresh broker_root**: `C2c_repo_fp.resolve_broker_root()`.
4. **Set env + re-exec**: `Unix.putenv "C2C_MCP_BROKER_ROOT" fresh_broker_root`,
   then `Unix.execvp Sys.argv.(0) Sys.argv` to re-run with same arguments.

**Why not `cmd_restart`?** `cmd_restart` would try to `kill` and `wait_for_exit`
the current outer process (deadlock). Instead, S2 inlines the orphan-capture +
execvp path directly.

### Files Changed (S2)

```
ocaml/c2c_start.ml  — exit-42 restart path in run_outer_loop
```
