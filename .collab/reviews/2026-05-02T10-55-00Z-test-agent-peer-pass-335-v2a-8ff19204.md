## Feature Commit: 43695999

### Core Logic

`nudge_tick` replaces `Broker.registration_is_alive` with `Broker.registration_liveness_state`:

```
Alive → nudge path (DND check + idle check + nudge_session)
Unknown → count via alive_no_pid if pid=None, otherwise no-op (don't send)
Dead → no-op (silently dropped)
```

**Correctness:**
- `Broker.registration_liveness_state` is the canonical tristate predicate — correct to use it ✅
- `Dead` → `()` (no nudge) ✅
- `Unknown` → `if reg.pid = None then incr alive_no_pid` (count pidless Unknown, no nudge) ✅
- `Alive` → full DND/idle/nudge path ✅
- **Deliberate divergence** from canonical `registration_is_alive` (which collapses Unknown→Alive for sweep/enqueue backward-compat) — intentional per design doc ✅
