## Blocking Bugs

### BUG 1: `--auto-join` flag is completely non-functional

**File**: `ocaml/cli/c2c.ml:7787` (and 7818, 7838)
**Severity**: BUG, **blocking**

`auto_join_rooms_opt` is parsed at line ~8020 and passed to `run_ephemeral_agent` at line ~8034. However, `start_cmd_args` is built at line ~7787 as:
```ocaml
Printf.sprintf "%s start %s -n %s --kickoff-prompt-file %s"
  (q c2c_bin) (q client) (q name) (q kickoff_path)
```
`auto_join` is **never included** in the `c2c start` shell command for any of Pane (line 7799), Background (line 7818), or Headless (line 7838) modes.

Additionally, `c2c start` has **no `--auto-join` CLI flag** — there is no mechanism to pass it through even if you wanted to.

The computed `auto_join` value is silently discarded. Spec D5 and D8 explicitly require `c2c agent run [--auto-join <rooms>]` to work.

---

### BUG 2: `C2C_MCP_AUTO_JOIN_ROOMS` not suppressed for `c2c agent run`

**File**: `ocaml/c2c_start.ml:936`
**Severity**: BUG, **blocking**

Per spec D5, ephemeral launch must NOT set `C2C_MCP_AUTO_JOIN_ROOMS` unless explicitly passed. In `build_env`, when `auto_join_rooms_override=None` (as it is for `c2c agent run`), `rooms_base` defaults to `"swarm-lounge"` and `C2C_MCP_AUTO_JOIN_ROOMS=swarm-lounge` is set in the child's env.

This means `c2c agent run` tmux subprocesses always inherit `C2C_MCP_AUTO_JOIN_ROOMS=swarm-lounge`. The no-autojoin-by-default requirement is violated.

For comparison, the `c2c start` path correctly threads `auto_join_rooms_override` — but `run_ephemeral_agent` always passes `auto_join_rooms_opt:None`.

## Non-Blocking Observations

### STYLE 1: `pane_arg` in `agent_refine_term` is dead code

**File**: `ocaml/cli/c2c.ml:7885`

`pane_arg` is defined but **never used** in the function body — only `reply_to_opt` is referenced. The function always passes `mode:Pane` hardcoded regardless of any `--pane` flag.

Harmless but misleading dead UI.

### DESIGN 1: `reply_to` embedding vs env-var threading inconsistency

`c2c start --reply-to` correctly threads `C2C_MCP_REPLY_TO` as an env var. `c2c agent run --mode pane` embeds the reply-to alias **only** in the kickoff template text — `start_cmd_args` doesn't include `--reply-to`. Works via template text, but inconsistent with spec D4 intent.

### OBS 1: Watchdog `Unix.kill 0` edge case

If `timeout=0` were passed, `start_watchdog ()` returns `0`, and `Unix.kill 0 Sys.sigterm` would send SIGTERM to the caller's process group. Timeout defaults to 1800s; unreachable in practice.

### OBS 2: Headless routing

`if client = "claude" || client = "opencode" then "codex-headless" else client` — correct per spec D3.

### OBS 3: Watchdog boot grace unified to 60s

`let boot_grace_deadline = start_time +. 60.0` — correct per spec D6.
