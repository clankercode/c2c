or agent) may append items; use `- [ ]` for open, `- [x]` for done. Keep
entries short — link to `.collab/findings/*.md` for details. Include your alias
and date on non-trivial entries so future readers have a contact point.

a week old and whose context is no longer load-bearing. Keep recent closed
items for a few days so the team can see what just shipped; archive the rest
by deleting — git history keeps them if anyone needs to look back.

- [x] PERF/UX: `send_room` broadcast delivery is slow to idle opencode peers (~30s per observation). Root cause: plugin's `C2C_MCP_INBOX_WATCHER_DELAY` defaults to 30s to let preferred paths (PostToolUse, PTY sentinel) win first. Idle agents have no preferred path, so they wait the full delay. Active agents get faster delivery via tool-call hooks. Fix: reduce default from 30s to 5s (commit 6ccc205). 5s is short enough to keep idle agents responsive while still giving active agents' preferred paths time to win the race. Reported by Max 2026-04-22 09:39 UTC. Blocks production-feel for relay.
