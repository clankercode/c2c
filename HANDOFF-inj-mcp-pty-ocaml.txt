### Key Discovery

Claude Code does NOT declare `experimental.claude/channel` in its initialize — so `client_supports_claude_channel` always returns `false` and auto-drain never fires. The fix is two-fold:
1. Server (c2c) declares the capability (not just detecting client support) ✓
2. Continuous delivery via inbox watcher (not just post-initialize drain) ✓

### Key Environment Variables

- `C2C_MCP_AUTO_DRAIN_CHANNEL` — default `0`, set to `1` to enable post-initialize drain (won't work without client declaring channel support)
- `C2C_MCP_CHANNEL_DELIVERY` — new flag being added (should default to `1`)
- `C2C_MCP_BROKER_ROOT` — broker directory (default: `.git/c2c/mcp`)
- `C2C_MCP_SESSION_ID` — session identifier
- `C2C_MCP_AUTO_REGISTER_ALIAS` — stable alias
- `C2C_MCP_AUTO_JOIN_ROOMS` — default `swarm-lounge`

### Known Limitations

- SSH sessions: PTY master not accessible server-side — MCP or history injection only
- History injection visible only in transcript, not in SSH chat UI
- Claude Code itself never declares `experimental.claude/channel` — so server must declare capability

## OpenCode Research Result

Saved: `docs/c2c-research/opencode-channel-notification.md`

- `notifications/claude/channel`: **NOT supported**
- Equivalent: `POST /tui/show-toast` (ephemeral toast, 5s, not in message history)
- Session message API (`POST /session/{id}/message`): sends user message, triggers AI response
- `chat.message` hook: read-only/transformative, cannot inject standalone messages
- Custom plugin bridge possible but requires custom dev

## Codex Research Result

Saved: `docs/c2c-research/codex-channel-notification.md`

- `notifications/claude/channel`: **NOT supported**
- No PostToolUse hook, no `experimental.claude/channel` capability
- Delivery via background daemon + PTY sentinel (`<c2c event="message_pending">poll mcp__c2c__poll_inbox</c2c>`) — not visible in chat
- Messages only appear in tool results, not native chat UI
- No plugin/hook system like Claude Code
